NOTE
+ BrowserRouter
  - เอาไว้ดู url เฉพาะหลังจาก localhost:3000
  - ต้องมีโครงสร้าง div หุ้ม component ไว้อีกที
  - รับ url มาจาก history component (background)
  - ตอน deploy ไปใน server อย่าลืม config url path 
    - server ปกติ จะ request - response กรณีหา url ไม่เจอจะส่ง 404
    - react server ถ้าหา url ไม่เจอจะส่ง index.html page ไม่ได้ส่ง 404 ดังนั้นตอน deploy ต้อง config อีกที

+ Route (<Route path="/" component={PageOne} />)
  - Route จะสนใจ path หลัง url เท่านั้น เช่น /, /page2, 
  - ถ้ามี path เหมือนกัน 2 component ก็จะ render ซ้อนกันสองอัน
  - exact หมายถึงต้อง extractedPath === path เท่านั้นถึงจะแสดงผล
  - ถ้าไม่มี exact และ extractedPath = /page จากนั้น path="/page/1" ทุกๆ  path ที่มีคำว่า /page/ จะแสดงหน้าจอซ้อนกัน เพราะว่าเบื้องหลังของ path คือ extractedPath.contains(path)
  - ใช้ <a></a> ในการเปลี่ยน route ไม่ดีเพราะว่าคลิกทีนึงจะถูกส่ง server แล้ว server response index.js , rerender index.html ใหม่ execute bundle.js ใหม่ทุกครั้ง ใช้ Link แทน

+ Link (react-router-dom) component
  - ใช้แทน <a></a> เมื่อ load page มีความหมายเดียวกับ a tag
  - <Link to="/">Back to page One</Link>
  - ป้องกัน (prevent) ไม่ให้ browser fetching new index.html แต่ url change
  - history มาดู url แล้วส่งให้ BrowserRouter
  - ห้ามใช้ <Link /> นอก <BrowserRouter> ดังนั้นทุก component ที่มี <Link /> จะต้องอยู่ภายใต้ BrowserRouter
